# Demo 使用指南

> 本指南告诉你：**在哪里点 → 看到什么 → 这个功能解决什么问题**。
> 全程基于「小王子」第 1 章演示，所有段落引用都是真实文本。

---

## 启动（3 步，约 5 分钟）

```bash
# 1. 进项目目录
cd docs/fhytoru/project

# 2. 配 env（重要：填入 MiniMax key）
cp .env.example .env.local
# 用编辑器打开 .env.local，把 MINIMAX_AUTH_TOKEN / MINIMAX_PAYGO_KEY / MINIMAX_BASE_URL 填好

# 3. 装依赖 + 跑生产服务器
npm install
npm run build     # 等约 30 秒，看到 ✓ Compiled successfully 即可
npm run start     # 看到 ▲ Next.js 16.2.6 Local: http://localhost:3000
```

**浏览器打开 `http://localhost:3000`**。

> 💡 **桌面浏览器请按 F12 → 切换设备模式 → 选 iPhone 14 (390×844)**。这套 UI 是按移动端 ≤430px 设计的，桌面全屏会拉得很难看。
>
> ⚠️ **不要用 `npm run dev`**。Next.js 16 在 Windows 上 dev 模式有 jest-worker 进程崩溃 bug，路由会随机 500。`build && start` 是稳定的生产模式。

## 4️⃣ 阅读器 `/book/little-prince/read`（**最核心**）

### 4.1 基础阅读
**看到什么**：
- 顶栏：返回 ‹ / 书名+章节 / 🤖 AI / 🗺️ 地图 / ⚙️ 设置
- 正文：宋体大字，行高 1.85，左右 padding 24px
- 第一段：「六岁那一年，我在一本描写原始森林的书里看见一幅扣人心弦的图画……」
- 底栏：阅读进度条 + 「AI 对话」圆形按钮
- 滚动到本章末尾，会冒出「本章结束 · 进入下一章」橙色按钮

**滚动行为**：
- **Lenis 平滑滚动**（不是浏览器默认的滚轮）
- 向下滚 → 顶栏自动收起，沉浸阅读
- 向上滚 → 顶栏滑出来
- 滚动时阅读位置自动写入 store（刷新后能回到原位）

### 4.2 长按段落菜单（**演示重头戏**）

**操作**：
- 📱 **移动端**：按住任意段落 ≥ 500ms
- 🖥️ **桌面端**：**右键单击**任意段落（更快）或按住鼠标 500ms

弹出菜单 4 个选项：

#### 🎨 生成画面 — AI 图像生成
- 点击 → 等 2.5s（"为上文搭一座想象的桥…" 等待动画）
- 调用 `/api/generate-image` → MiniMax Image-01 模型
- **正常情况**：约 10-15s 后弹出生成的插图（3:4 竖图），可点「使用此图」插入段落下方
- **额度不足时**：自动 fallback 到 3 个 emoji 调色板（🌹/🦊/✨/🪐/🐍 随机选 3），**顶部显示琥珀色警告**：「⚠️ AI 图像额度不足，已切换为本地候选画板」
- 选中后图片永久插入段落下方（写入 Zustand store）

**这是什么用的**：把抽象文字变成视觉记忆，特别适合景物描写、情感转折段。

#### 🤖 问 AI — 流式对话
- 点击 → 底部弹出 ChatDrawer（高度可拖拽 35%-90%）
- 输入框预填了段落引用，可直接发问
- 调用 `/api/chat` → MiniMax LLM 流式返回（边吐字边显示）
- **真实实测**：输入「这一段为什么提到丛林？」，AI 回答：「这段提到丛林，是因为它承载了'我'对远方世界的好奇与想象。正是在那片神秘的丛林里，蟒蛇囫囵吞下猎物的奇事让'我'念念不忘……」

**这是什么用的**：随读随问，AI 知道你读到哪一段，回答不会剧透后文。

**进阶**：菜单里的「🤖 问 AI」按钮**需要长按 450ms**才触发（防止误触），普通点击没反应。

#### 📌 标记 — 段落书签
- 点击 → 段落左侧出现**主色（橙）竖线**标记
- 写入 localStorage（key: `sanweishuwu-app`，字段 `bookmarksByBook`）
- 刷新页面后书签仍在
- 再次长按该段落 → 菜单变成「📌 取消标记」，点击移除

**这是什么用的**：标记触动你的段落。比如标记「我画的不是一顶帽子，而是一条正在消化大象的蟒蛇啊。」之后能在阅读地图里看到所有书签。

#### 🎭 广播剧朗读 — 多角色 TTS
- **必须长按含引号的段落**！例如：
  - 段落 `p-c1-5`：「他们回答我说：'一顶帽子有什么可怕的？'」
  - 段落 `p-c1-2`：「书中说：'蟒蛇囫囵吞下猎物……'」
- 点击 → 弹出全屏广播剧界面，三角色头像：🎧 叙述者 / 🤴 小王子 / 🌹 玫瑰
- 点中间播放按钮 → 调用 `/api/generate-tts` → MiniMax `speech-2.8-hd`
- 多 voice 分配（叙述者一种音色、对话角色另一种音色）
- 正常情况：自动按句切换角色头像高亮 + 播放对应音频
- **额度不足时**：显示琥珀色警告「⚠️ AI 语音额度不足，本段暂时无法朗读」，仍可关闭

**这是什么用的**：把对话密集的段落变成有声小剧场，特别适合通勤听书。

### 4.3 阅读设置抽屉

**操作**：顶栏 ⚙️ → 弹出全屏设置抽屉

**可调**：
- **阅读模式**：标准阅读 / AI 沉浸朗读（大字 + 自动锚点推进）
- **朗读速度**：0.75x / 1x / 1.25x / 1.5x
- **字号**：14 / 16 / 18 / 20 / 22 px
- **亮度**：滑块（影响整页 `filter: brightness()`）
- **主题**：明亮 / 暗黑 / 跟随系统
- **Voice profile**：青瓷 / 松烟 / 琥珀（AI 朗读用的音色预设）

所有设置 → Zustand persist 到 localStorage，下次打开仍生效。

8. **完成页**（30s）— `/book/little-prince/finished` 展示情感曲线 + 仪式感收尾

**总计约 3 分钟**，留 2 分钟回答问题。

## 故障排查

| 症状 | 原因 | 解决 |
|------|------|------|
| `npm run dev` 报 `Jest worker encountered ... exceptions` | Next.js 16 在 Windows 上 dev 模式有 bug | 用 `npm run build && npm run start` |
| 打不开 `http://localhost:3000` | 之前的 dev server 进程没退 | `taskkill /F /IM node.exe`，重跑 |
| 长按菜单不弹 | 桌面浏览器选文字干扰 | 用**右键单击**段落代替长按 |
| 「问 AI」点了没反应 | 它需要长按 450ms（防误触） | 在「🤖 问 AI」按钮上按住不动 |
| 「生成画面」转圈很久 | 调真实 API 需 10-15s | 等等就好；额度不足会显示警告并 fallback |
| 「广播剧」没声音 | TTS 额度耗尽 | 看顶部琥珀色警告 |
| 章节封面无 BGM | MiniMax 音乐生成额度不足（已知） | 不修，UI 已 graceful fallback |
| 显示「未找到书籍」 | 你访问了 yugong 等没数据的书 | 只演示 `little-prince`，其他 3 本只有元数据 |

## 关于本快照的技术亮点

如果评委问 "你做了什么"，可以提：

1. **三人融合**：Member 1 的 UI 设计（802 行 OKLCH 设计系统、3D 书架）+ Member 2 的 AI 工程（MiniMax 流式 LLM 网关、本地 LocalAiProvider）+ Member 3 的多模态（Python pipeline 生成插图 + TTS）
2. **修复了 5 个关键 bug**：
   - Next.js 16 Turbopack panic on globals.css → 改 webpack
   - `"use client"` 误标导致 Server Component 调用 500
   - 长按 `select-text` 干扰阈值 → 改 `select-none` + 加 `onContextMenu` 兼容桌面右键
   - 书签按钮从 `console.log` 占位符改为真实 Zustand 持久化
   - httpx `verify=False` 复用导致 Python pipeline "Cannot open client more than once"
3. **加了 UX 保护**：API 额度不足时显示琥珀色警告条，不让演示"哑掉"
